# AGENTS

This file is reserved for instructions intended for automation agents/tools.

## Language Policy

All project artifacts must be written in English:

- Directives
- Documentation
- Code (identifiers when newly introduced)
- Comments

## Engineering Best Practices

- Keep business logic, hardware access, and presentation concerns separated.
- Prefer small, single-responsibility modules with clear public APIs.
- Keep shared domain types centralized to avoid duplication and cyclic dependencies.
- Use meaningful names and avoid abbreviations that reduce readability.
- Favor composition over global state when wiring runtime components.
- Keep comments concise and explain intent, not obvious implementation details.
- Validate changes with a build before considering a task complete.
